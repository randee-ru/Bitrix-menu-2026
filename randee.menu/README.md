# Randee Menu (`randee.menu`)

Модуль **1С-Битрикс** для централизованного управления **несколькими многоуровневыми меню** сайта из административной панели: коды меню, вложенность без ограничения глубины, сортировка, активность, ссылки и дополнительные параметры в JSON.

**Версия:** см. `install/version.php` (текущая ветка — 1.0.2).

## Возможности

| Возможность | Описание |
|-------------|----------|
| Несколько меню | Отдельные сущности с **уникальным кодом** (например `main`, `mobile`, `heads`) |
| Вложенность | Любая глубина через **родительский пункт** (`PARENT_ID`, уровень `DEPTH_LEVEL` в БД) |
| Активность | Пункты и меню можно отключать без удаления |
| Сортировка | Числовое поле **SORT**; на странице пунктов — **полное дерево** и **drag-and-drop** с сохранением порядка и вложенности |
| Ссылки | Внутренние (`/path/`) и внешние (`https://…`), тип `inner` / `outer` |
| Target | `_self`, `_blank`, `_parent`, `_top` |
| Параметры | Поле **JSON** для атрибутов ссылки (`class`, `data-*` и т.д.) |

## Административный интерфейс

### Где находится пункт меню

В левой колонке админки: **«Сервисы» → «Меню сайта»** (файл `admin/menu.php`, `parent_menu` = `global_menu_services`).

### Страницы (`/bitrix/admin/`)

| Файл | Назначение |
|------|------------|
| `randee_menu_menus.php` | Список меню: код, название, сортировка, активность. Контекстное меню строк: редактирование, пункты, удаление |
| `randee_menu_edit.php` | Добавление / редактирование меню (код, название, описание, сортировка, активность). Кнопка перехода к пунктам |
| `randee_menu_items.php` | **Полное дерево** пунктов выбранного меню (`MENU_ID`). **Перетаскивание** за маркер слева; кнопка **«Сохранить порядок (вложенность)»** отправляет структуру на сервер и обновляет `PARENT_ID` и `SORT` |
| `randee_menu_item_edit.php` | Добавление / редактирование одного пункта |

Типовые URL (с учётом `lang`):

- Список меню: `randee_menu_menus.php?lang=ru`  
- Редактирование меню: `randee_menu_edit.php?ID=1&lang=ru`  
- Пункты: `randee_menu_items.php?MENU_ID=1&lang=ru`  
- Пункт: `randee_menu_item_edit.php?MENU_ID=1&ID=…&lang=ru`

### Дерево и drag-and-drop

1. Откройте **Пункты меню** для нужного меню.  
2. Отображается **всё дерево** сразу (не по одному уровню).  
3. Тяните строку за иконку **⠿**; можно менять порядок среди соседей и вкладывать пункты друг в друга.  
4. Нажмите **«Сохранить порядок (вложенность)»** — изменения уходят AJAX-запросом (`action=reorder_tree`), на сервере вызывается `MenuManager::applyAdminTreeOrder()`.  
5. Поле **DEPTH_LEVEL** у пунктов пересчитывается ORM при смене родителя.

Скрипты подключаются с **`/local/modules/randee.menu/js/`** (при отсутствии — резервный путь `/bitrix/modules/randee.menu/js/`).

## База данных

### `b_randee_menu`

| Поле | Описание |
|------|----------|
| ID | Первичный ключ |
| CODE | Уникальный код меню (латиница, до 50 символов) |
| NAME | Название |
| DESCRIPTION | Описание (text) |
| SORT | Сортировка в списке меню |
| ACTIVE | `Y` / `N` |
| DATE_CREATE, DATE_MODIFY | Служебные даты |

### `b_randee_menu_item`

| Поле | Описание |
|------|----------|
| ID | Первичный ключ |
| MENU_ID | Привязка к меню |
| PARENT_ID | Родитель (`0` — корень) |
| NAME | Текст пункта |
| LINK | URL (до 500 символов) |
| LINK_TYPE | `inner` / `outer` |
| SORT | Порядок среди соседей |
| ACTIVE | `Y` / `N` |
| PARAMS | JSON-текст |
| TARGET | `_self`, `_blank`, `_parent`, `_top` |
| DEPTH_LEVEL | Уровень вложенности |
| DATE_CREATE, DATE_MODIFY | Служебные даты |

Скрипты создания и удаления таблиц: `install/db/mysql/install.sql`, `uninstall.sql`.

## Установка и обновление

### 1. Копирование модуля

Рекомендуется:

```text
/local/modules/randee.menu/
```

Скопируйте каталог `randee.menu` целиком в указанный путь.

### 2. Установка через админку

1. **Настройки → Настройки продукта → Модули** (в зависимости от редакции путь может называться иначе).  
2. Найдите **Randee Menu** / `randee.menu`.  
3. Нажмите **Установить**.

Установщик:

- регистрирует модуль и выполняет SQL из `install.sql`;  
- копирует PHP из `install/admin/` в **`/bitrix/admin/`**;  
- копирует компонент в **`/local/components/randee.menu/`**;  
- копирует **`js/`** в **`/local/modules/randee.menu/js/`** (если каталог модуля в `local` совпадает с целевым).

### 3. Обновление файлов вручную

Если вы правите код в репозитории и нужно обновить только админ-скрипты на сервере, скопируйте файлы из `install/admin/` в `/bitrix/admin/` и при необходимости обновите `lib/`, `js/` внутри `local/modules/randee.menu/`.

## Работа с меню в админке

1. **Сервисы → Меню сайта** — создайте меню (код латиницей, уникальный).  
2. Через контекстное меню или кнопки откройте **Пункты меню**.  
3. Добавляйте пункты (**Добавить пункт**), задавайте родителя, ссылку, сортировку.  
4. Для изменения иерархии и порядка используйте дерево и **Сохранить порядок** на странице пунктов.

**Код меню** после создания при редактировании отображается **только для чтения** (readonly), чтобы не нарушать привязки в шаблонах и компоненте.

### JSON в пункте

Пример:

```json
{"class": "nav-link", "data-bs-toggle": "dropdown"}
```

Атрибуты попадут на тег ссылки в шаблоне компонента (см. `template.php`).

## Вывод на сайте (компонент)

Подключение:

```php
<?php
$APPLICATION->IncludeComponent(
    'randee.menu:menu',
    '.default',
    [
        'MENU_CODE'    => 'main',   // код меню из админки
        'ACTIVE_ONLY'  => 'Y',      // только активные пункты
        'CACHE_TIME'   => 3600,
        'CACHE_GROUPS' => 'N',
    ]
);
?>
```

Параметры задаются также в `.parameters.php` компонента (`MENU_CODE`, `ACTIVE_ONLY`, кеш).

### Шаблон

Копирование для правок:

```text
/local/templates/ВАШ_ШАБЛОН/components/randee.menu/menu/.default/
```

Или создайте именованный шаблон и укажите его вторым аргументом `IncludeComponent`.

## Программный API

```php
\Bitrix\Main\Loader::includeModule('randee.menu');

use Randee\Menu\MenuManager;

// Дерево: каждый элемент содержит вложенный массив CHILDREN
$tree = MenuManager::getMenuTree('main', true); // второй аргумент — только активные

// Плоский список пунктов
$flat = MenuManager::getMenuItems('main', true);

// Карточка меню по коду или ID
$menu = MenuManager::getMenu('main', false);
```

Дополнительно (в основном для админки и интеграций):

- `MenuManager::getAdminTree(int $menuId)` — полное дерево пунктов меню, включая неактивные.  
- `MenuManager::applyAdminTreeOrder(int $menuId, array $nodes)` — применение дерева после drag-and-drop (`$nodes` — вложенные массивы с ключами `id`, `children`).  
- `MenuManager::clearCache(string $menuCode = '')` — сброс файлового кеша компонента по тегу.

Прямой доступ к ORM:

```php
use Randee\Menu\MenuTable;
use Randee\Menu\MenuItemTable;

$row = MenuTable::getRowById(1);
$list = MenuItemTable::getList(['filter' => ['MENU_ID' => 1], 'order' => ['SORT' => 'ASC']]);
```

## Кеш и сброс

Компонент кеширует результат по коду меню и параметрам.

Важно: в используемой версии Битрикса `Bitrix\Main\Data\ManagedCache` может **не содержать** метода `initCache()`. Поэтому в `randee.menu:menu` кеш реализован через `ManagedCache::read() / get() / set()`.

После изменений в админке при сохранении дерева вызывается `MenuManager::clearCache()` для соответствующего кода. При массовых правках в БД вручную можно вызвать `clearCache()` из своего кода или очистить кеш в **Настройки → Производительность**.

## Перенос и резервное копирование

1. **Файлы:** каталог `local/modules/randee.menu` (и при необходимости скопированные `bitrix/admin/randee_menu_*.php`, `local/components/randee.menu`).  
2. **БД:** экспорт таблиц `b_randee_menu` и `b_randee_menu_item` (`mysqldump` или средства хостинга).  
3. На новом сайте разместите модуль, при чистой БД выполните **установку модуля**, затем **импорт** таблиц (или импортируйте данные в уже созданные таблицы с тем же префиксом).

Если на целевом сайте модуль уже установлен и таблицы совпадают по структуре, достаточно импорта данных и обновления файлов.

## Удаление модуля

**Настройки → Модули** — удаление модуля `randee.menu`.

Удаляются (согласно `UnInstallDB` / `UnInstallFiles`):

- таблицы `b_randee_menu`, `b_randee_menu_item`;  
- файлы `randee_menu_*.php` из `/bitrix/admin/`;  
- каталог **`/local/components/randee.menu/`**.

Каталог **`/local/modules/randee.menu`** при необходимости удалите вручную. Скрипты в `local/modules/randee.menu/js/` входят в состав модуля и удаляются вместе с папкой модуля.

## Требования

- **1С-Битрикс** с поддержкой **D7** (ORM, `Bitrix\Main\Loader`).  
- **PHP 8.2+** (рекомендуется для соответствия современным ядрам и окружениям).  
- **MySQL 5.7+** / **8.0** или совместимая MariaDB.

## Устранение неполадок

| Симптом | Что проверить |
|---------|----------------|
| Пустые поля на `randee_menu_edit.php` при `?ID=…` | Обновите файлы админки из `install/admin/`; в форме должны быть скрытые поля `ID` / `MENU_ID` при редактировании; используйте полный перезапуск страницы (Ctrl+F5). |
| Ошибка при сохранении дерева «не хватает пунктов» | Обновите `menu_admin_tree.js` и `sortable.min.js` в `local/modules/randee.menu/js/` — сериализация должна идти от корневого `<ol>`, а не от обёртки-`div`. |
| 500 на странице с компонентом, текст `Call to undefined method Bitrix\Main\Data\ManagedCache::initCache()` | Это лечится обновлением `components/randee.menu/menu/component.php` (кеш должен делаться через `ManagedCache::read/get/set`). |
| Не подсвечивается нужный раздел админки | Убедитесь, что в `admin/menu.php` указан нужный `parent_menu` (`global_menu_services` для «Сервисы»). |
| Компонент не находится | Проверьте наличие `/local/components/randee.menu/menu/` и вызов `Loader::includeModule('randee.menu')` не требуется перед IncludeComponent, если ядро подключает модуль автоматически — при ошибке подключите модуль явно. |
| Кеш старого меню | `MenuManager::clearCache('код_меню')` или очистка управляемого кеша в админке. |

## Лицензия и автор

Модуль разработан в рамках проекта Randee. Условия распространения задайте в соответствии с политикой вашего продукта.

При публикации на продакшене не оставляйте на сервере служебные скрипты восстановления (`restore.php`) и резервные архивы в общедоступном каталоге сайта.
